Fix critical buffer overflows and display driver bugs across UART, DTMF, and ST7565

This commit applies 6 production-ready patches to address critical stability issues
and improve code robustness:

CRITICAL FIXES (🔴 High Impact):
- uart.c: Replace unsafe strcpy() with bounds-checked strncpy() for external UART
  version requests. Prevents SEGFAULT when version string exceeds 16-byte buffer.
- generic.c: Replace unsafe strcpy() with bounds-checked strncpy() for DTMF user
  input. Prevents memory corruption on long DTMF sequences.
- st7565.c: Fix FillScreen() algorithm - was using incorrect DrawLine() pattern
  instead of proper memset(). Corrupted display clear/fill operations.
- st7565.c: Add missing CS_Release() in ContrastAndInv(). Prevents SPI bus lockup
  when adjusting display contrast/inversion in menu.
- st7565.c: Add missing #include <string.h> for memset() declaration.

OPTIMIZATION & CLEANUP (🟡 Low Impact):
- board.c: Wrap GPIO_PIN_13 initialization in #ifdef ENABLE_FLASHLIGHT. Saves
  ~50 bytes when flashlight feature is disabled.
- adc.c: Add unsigned suffix to bit-shift operation and clarify comment. Improves
  compiler optimization and code intent clarity.

TESTING:
- All 7 firmware variants build successfully without errors
- Binary sizes remain well within 118KB flash constraint:
  * ApeX: 82KB (70%)
  * Bandscope: 73KB (62%)
  * Basic: 74KB (63%)
  * Broadcast: 71KB (60%)
  * Game: 72KB (61%)
  * RescueOps: 68KB (58%)

EXPECTED IMPROVEMENTS:
✅ Prevents 2 critical buffer overflow crashes
✅ Eliminates SPI bus lockup on display operations
✅ Fixes display rendering algorithm
✅ Improves UART/external tool compatibility (Chirp, UVTools)
✅ Saves ~50 bytes with conditional compilation
